## 1. 내 프로젝트는 어떤 유형인가요

이 프로젝트는 Streamlit으로 만든 앱입니다. `app.py` 파일에서 `import streamlit as st`를 사용하고 있고, `requirements.txt`에도 `streamlit`이 들어 있습니다. Streamlit은 데이터 분석 화면이나 데모 앱을 쉽게 만드는 도구입니다. 그래서 이 프로젝트는 **E. 데이터/데모 앱**으로 분류합니다.  
또한 `opencv-python-headless`, `mediapipe`, `plotly`, `supabase`를 사용합니다. 즉 웹캠 영상 처리, 손가락 뼈대 분석, 3D 그래프 표시, 클라우드 데이터 저장 기능이 함께 있는 앱입니다.

## 3. 사용할 서비스와 이유

### 3-1. 배포 서비스: Streamlit Community Cloud
- 이걸 고른 이유: 이 프로젝트가 Streamlit 앱이고, `app.py` 하나를 중심으로 실행됩니다. Streamlit Community Cloud는 GitHub 저장소를 연결하면 Streamlit 앱을 쉽게 배포할 수 있습니다.
- 무료로 되나요: 됩니다. 공식 문서에서 Streamlit Community Cloud는 Streamlit 앱을 무료로 만들고 배포하고 관리할 수 있다고 안내합니다. 확인 날짜: 2026-07-22. https://docs.streamlit.io/deploy/streamlit-community-cloud
- 무료의 한계(솔직하게): 앱이 무거우면 느릴 수 있습니다. 이 프로젝트는 `mediapipe`와 `opencv`처럼 무거운 라이브러리를 쓰므로 처음 켜질 때 시간이 오래 걸릴 수 있습니다. 또한 현재 웹캠 코드는 배포 서버의 카메라를 찾는 방식이라, 배포 후 방문자의 웹캠이 바로 동작하지 않을 가능성이 큽니다.
- 유료로 가면: Streamlit Community Cloud 자체의 간단한 유료 업그레이드보다, 이 앱은 먼저 웹캠 구조를 고치는 것이 더 중요합니다. 꼭 서버를 직접 운영해야 한다면 Render 같은 서버 서비스가 대안이지만, 무료 서버는 잠들 수 있고 유료는 보통 월 약 7달러 이상입니다.
- 카드 등록: 필요 없음

### 3-2. 추가 서비스: Supabase
- 이걸 고른 이유: 이미 코드에서 Supabase를 사용하고 있습니다. `app.py`에 `SUPABASE_URL`, `SUPABASE_KEY`, `motions` 테이블을 쓰는 코드가 있습니다.
- 무료로 되나요: 됩니다. Supabase 공식 요금표 기준 Free 플랜은 월 0달러입니다. 확인 날짜: 2026-07-22. https://supabase.com/pricing
- 무료의 한계(솔직하게): 무료 플랜은 프로젝트 2개 제한이 있고, 1주일 동안 사용하지 않으면 프로젝트가 멈출 수 있습니다. 데이터베이스 크기는 500MB까지 무료입니다.
- 유료로 가면: Pro 플랜은 월 25달러부터입니다. 무료 플랜보다 용량이 커지고, 프로젝트가 자동으로 멈추지 않는 점이 좋아집니다.
- 카드 등록: 무료 플랜만 쓰면 보통 필요 없음. 유료 플랜으로 올리면 필요함.

### 3-3. 고려했지만 선택하지 않은 대안

Render: Streamlit 앱도 올릴 수 있지만, Streamlit 전용 서비스보다 설정이 더 어렵고 무료 서버는 잠들 수 있습니다.  
Railway: 서버 앱 배포는 가능하지만 무료 사용량과 과금 정책을 신경 써야 해서 비개발자에게는 Streamlit Community Cloud보다 어렵습니다.

## 6. 배포 전에 고쳐야 할 부분

- 웹캠 기능: 현재 코드는 `cv2.VideoCapture(0)`을 사용합니다. 이것은 내 컴퓨터에서 실행할 때는 내 웹캠을 열지만, 인터넷 배포 후에는 배포 서버의 카메라를 찾게 됩니다. 그래서 방문자의 웹캠을 쓰려면 브라우저 웹캠 방식으로 고쳐야 합니다.
- 파일 저장: 현재 녹화 결과를 `data/` 폴더에 JSON 파일로 저장합니다. 배포 서버에서는 이 파일이 재시작 후 사라질 수 있습니다. 오래 보관해야 하는 데이터는 Supabase에 저장해야 합니다.
- 무거운 라이브러리: `mediapipe`, `opencv-python-headless`는 설치와 실행이 무거울 수 있습니다. 배포가 느리거나 실패하면 로그를 보고 조정해야 합니다.
- GitHub 반영: 현재 로컬 `app.py`에 아직 GitHub에 올리지 않은 변경이 있습니다. 실제 배포 전에 다시 커밋하고 푸시해야 합니다.

## 7. 배포 순서 요약

1. 로컬에서 앱이 정상 실행되는지 확인합니다.
2. 수정한 코드를 GitHub에 올립니다.
3. Streamlit Community Cloud에 GitHub 계정으로 로그인합니다.
4. `Suguri123/FMT_project` 저장소를 선택합니다.
5. 실행 파일을 `app.py`로 지정합니다.
6. Secrets 설정에 `SUPABASE_URL`, `SUPABASE_KEY`를 넣습니다.
7. Deploy 버튼을 눌러 배포합니다.
8. 배포된 인터넷 주소에 접속해서 화면이 뜨는지 확인합니다.

## 8. 예상되는 어려움과 대비책

| 예상 문제 | 이럴 때는 |
|---|---|
| 웹캠이 배포 사이트에서 안 켜짐 | 현재 방식은 배포 서버 카메라를 찾기 때문입니다. 브라우저 웹캠 방식으로 코드를 고쳐야 합니다. |
| 앱 설치가 오래 걸림 | `mediapipe`와 `opencv`가 무거워서 그럴 수 있습니다. 배포 로그를 확인합니다. |
| Supabase 데이터가 안 보임 | Streamlit Secrets에 `SUPABASE_URL`, `SUPABASE_KEY`가 정확히 들어갔는지 확인합니다. |
| 저장된 JSON 파일이 사라짐 | 배포 서버 파일은 오래 보관용이 아닙니다. Supabase 저장을 사용해야 합니다. |
| 카드 등록 화면이 나옴 | 진행하지 말고 멈춘 뒤 강사에게 문의합니다. |
