# 손가락 움직임 기록 및 재생 스트림릿(Streamlit) 서비스 기획서

## 1. 프로젝트 개요
- **프로젝트명**: FingerMotion Studio (가칭)
- **서비스 설명**: 사용자의 웹캠을 통해 손가락의 움직임을 실시간으로 인식하여 일정 시간 동안 기록하고, 저장된 데이터를 기반으로 손가락 움직임을 애니메이션 형태로 다시 재생할 수 있는 웹 기반 프로그램입니다.

## 2. 서비스 목적 및 기대 효과
- **목적**: 사람의 정밀한 손가락 움직임을 데이터화(X, Y, Z 좌표)하여 저장하고, 이를 필요할 때 언제든 동일한 동작으로 재현하는 것입니다.
- **활용 분야**:
  - **수어 학습 플랫폼**: 전문가의 수어 동작을 기록하고 학습자가 3D 또는 2D 애니메이션으로 다각도에서 돌려보며 학습.
  - **로봇 제어 데이터**: 로봇 손(Robotic Hand) 제어를 위한 정밀한 시계열 모션 데이터 구축.
  - **재활 치료 및 메타버스**: 손가락 재활 환자의 움직임 변화 추적 및 메타버스 아바타 모션 캡처.

## 3. 핵심 기능 (Core Features)
**[손가락 21개 관절 움직임 기록 및 애니메이션 재생 기능]**
1. **타이머 기반 기록 (Recording)**
   - 사용자가 원하는 기록 시간(예: 3초, 5초, 10초)을 설정할 수 있습니다.
   - 웹캠을 통해 실시간으로 손을 인식하고, 21개의 손가락 관절(랜드마크)의 프레임별 좌표 데이터를 수집합니다.
2. **데이터 저장 및 관리 (Saving)**
   - 기록이 완료된 시계열 모션 데이터를 JSON 또는 CSV 형태로 저장합니다.
3. **애니메이션 재생 (Playback)**
   - 저장된 데이터를 불러와서, 화면 상에 골격(Skeleton) 애니메이션 형태로 움직임을 다시 재생합니다.
   - 재생 속도 조절(배속/느리게) 기능을 지원하여 정밀한 움직임 확인이 가능합니다.

## 4. 기술 스택 (Tech Stack)
- **프론트엔드 및 웹 프레임워크**: `Streamlit` (빠른 UI 구성 및 파이썬 기반 웹 서비스 배포)
- **손가락 관절 인식 (Computer Vision)**: `MediaPipe Hands` (구글의 실시간 21개 손가락 랜드마크 추적 라이브러리) & `OpenCV` (웹캠 영상 처리)
- **데이터 시각화 및 애니메이션**: `Matplotlib` 또는 `Plotly` (3D/2D 애니메이션 렌더링), `Streamlit-WebRTC` (실시간 영상 스트리밍)
- **데이터 처리**: `Pandas`, `NumPy`

## 5. UI/UX 구성 (Streamlit 화면 설계)
서비스는 크게 3개의 패널 또는 탭으로 구성됩니다.

### Tab 1: 🎥 실시간 기록 (Record)
- **웹캠 화면**: 사용자의 실시간 영상 및 인식된 손가락 랜드마크(뼈대) 오버레이 출력.
- **컨트롤 패널**:
  - 기록 시간 설정 슬라이더 (예: 1초 ~ 15초)
  - `[▶ 기록 시작]` 버튼 및 남은 시간 카운트다운 표시.
- **결과 피드백**: 기록 완료 후 "데이터가 성공적으로 저장되었습니다" 메시지 및 다운로드 버튼.

### Tab 2: 🎬 모션 재생 (Playback)
- **데이터 로드**: 이전에 저장된 모션 데이터 목록 선택 또는 파일 업로드 기능.
- **애니메이션 뷰어**: 비디오 영상이 아닌, 추출된 좌표 데이터만으로 구성된 손가락 뼈대 애니메이션(2D 또는 3D) 출력.
- **컨트롤러**: 재생, 일시정지, 타임라인 슬라이더, 재생 속도 조절(x0.5, x1.0, x2.0).

### Tab 3: 📊 데이터 분석 (Data View)
- 선택된 프레임별 각 관절의 이동 궤적을 그래프로 확인. (예: 검지 손가락 끝의 이동 경로)

## 6. 개발 단계별 마일스톤
- **Phase 1 (프로토타입)**: MediaPipe를 이용한 웹캠 손가락 21개 좌표 실시간 추출 및 시각화.
- **Phase 2 (핵심 기능 구현)**: 타이머 기반 좌표 데이터 프레임별 저장(CSV/JSON) 및 Plotly/Matplotlib를 이용한 데이터 기반 애니메이션 재생 기능 구현.
- **Phase 3 (웹 서비스화)**: Streamlit과 Streamlit-WebRTC를 결합하여 웹 브라우저 상에서 동작하는 UI 구성 및 완성.

## 7. MVP 진단 및 개발 방향 재설정 (AI 리포트 반영)
본 프로젝트는 **'방향 명확 + 오버스펙'** 유형으로 진단되었습니다. 짧은 MVP 기간과 비개발자 환경을 고려하여 다음과 같이 개발 범위를 타협 및 재설정합니다.

- **유지 및 집중할 핵심 가치**: MediaPipe 기반의 정밀한 모션 좌표(X,Y,Z) 데이터 추출 및 로컬 JSON 저장 흐름 완벽 구현.
- **축소 (오버스펙 방지)**: 실제 3D 캐릭터 모델(아바타) 웹 렌더링은 현재 기술 스택(Streamlit)에서 난이도가 높으므로, 이번 단계에서는 **Plotly 기반의 3D 뼈대(Skeleton) 애니메이션 시각화**로 데이터 정확도를 검증하는 데 만족합니다.
- **클라우드 연동 (선택)**: 복잡한 회원가입 기반 DB 대신, 필요시 **Supabase**를 활용해 JSON 파일만 업로드/다운로드 하는 초간단 클라우드 저장소 형태로 구현합니다.
- **최종 목표 구체화**: "화려한 3D 아바타 재생"보다는 "로봇 공학자나 3D 애니메이터가 즉시 활용할 수 있는 정확한 모션 데이터를 클라우드에 구축하는 도구"로서의 가치에 집중합니다.
